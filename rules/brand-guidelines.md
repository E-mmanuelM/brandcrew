# Brand Guidelines — Visual Identity

> Load this file before designing any image. Apply every rule here.

---

## Customize Your Brand

This system is designed for YOUR personal brand. The default theme is a starting point — you should create your own.

### How to set up your brand colors and fonts

1. Open `templates/themes/default.json` — this is the active theme
2. Change the colors, fonts, and style to match YOUR brand identity
3. Or create a new theme file (e.g., `templates/themes/my-brand.json`) and update the reference in `scripts/render_template.py`

### What goes in a theme file

A theme file is a simple JSON with CSS variables. Here's the structure:

```json
{
  "name": "Your Brand Name",
  "description": "Short description of your visual style",
  "font_import": "@import url('https://fonts.googleapis.com/css2?family=YOUR+HEADLINE+FONT&family=YOUR+BODY+FONT&display=swap');",
  "css_variables": {
    "--bg-cream": "#YOUR_BACKGROUND_COLOR",
    "--forest": "#YOUR_PRIMARY_COLOR",
    "--forest-deep": "#YOUR_PRIMARY_DARK",
    "--sage": "#YOUR_SECONDARY_COLOR",
    "--accent-lilac": "#YOUR_ACCENT_COLOR",
    "--accent-coral": "#YOUR_ALT_ACCENT_COLOR",
    "--text-on-dark": "#YOUR_LIGHT_TEXT",
    "--text-on-light": "#YOUR_DARK_TEXT",
    "--text-muted": "#YOUR_MUTED_TEXT",
    "--card-radius": "24px",
    "--inner-radius": "16px",
    "--font-display": "'Your Headline Font', Georgia, serif",
    "--font-body": "'Your Body Font', -apple-system, sans-serif"
  }
}
```

### Quick start: Ask AI to help

Open the project in Claude Code and say:
> "Help me create a brand theme for my LinkedIn content. My brand colors are [describe your style] and I want it to feel [describe the vibe]."

The AI will generate a complete theme JSON file for you.

### What to think about when choosing your brand

- **2-3 core colors** — a background, a primary, and an accent. Less is more.
- **Headline font** — bold, editorial fonts stop the scroll. Browse [Google Fonts](https://fonts.google.com/) for options.
- **Body font** — clean and readable. Sans-serif works best for LinkedIn.
- **Vibe** — are you editorial and polished? Minimal and modern? Bold and energetic? Your colors and fonts should reflect that.

## Image System — HTML/CSS Templates + Playwright

Images are generated via HTML/CSS templates rendered to JPG by Playwright on VPS.

- Templates live in `templates/*.html`
- Theme colors/fonts are in `templates/themes/default.json` (CSS variables)
- The Social Media Designer Agent picks a template, fills variables, and calls `scripts/render_template.py`
- Output: 1080×1350 JPG at 1x scale (~80-150KB)

---

## Design Philosophy

Images look like pages torn from a premium editorial magazine — not generic AI infographics. They have depth, texture through layering, concrete visual metaphors, and typographic confidence. Every image should feel hand-designed by a creative director, not auto-generated by a template.

**The three pillars:**
1. **Visual metaphor first** — every concept gets a concrete, tangible visual (a basket catching coins, a calendar tracking streaks, a net with holes). Never just text on shapes.
2. **Layered depth** — nested cards, overlapping panels, foreground/background separation. Images feel like they have physical layers, not flat stickers.
3. **Editorial typography** — bold, opinionated type that carries meaning on its own. The headline IS the design.

---

## The ICS Rule — Every Image Must Have All Three

Before designing any image, confirm you have all three:

| I | **Image type** | What template is this? (editorial data card, transformation, split comparison, magazine infographic, bold text + metaphor, quote card) |
| C | **Content** | What specific text, data, steps, or labels appear in the image? List them explicitly. |
| S | **Style** | What is the visual treatment? The template handles layout, but what content fills it? |

## Default Color Palette

These are the default theme colors included with BrandCrew. Customize them in `templates/themes/default.json` to match your brand.

| Role | Name | Hex | Usage |
|---|---|---|---|
| Background | Warm cream | `#F0EBE0` | Always the outer canvas — every image |
| Primary element | Forest green | `#2D4A3A` | Cards, panels, chart bars, frames, deep surfaces |
| Secondary fill | Sage green | `#6BB89A` | Accents, labels, small UI elements, category tags |
| Accent pop | Soft lilac | `#C9A8D4` | ONE highlight per image max — a single word, one data point |
| Accent alt | Warm coral | `#D4836B` | "Bad" side of comparisons, crossed-out elements |
| Text on dark | Off-white | `#EDE6D8` | Text sitting on dark backgrounds |
| Text on light | Dark green | `#1E2E1E` | Text sitting on light backgrounds |
| Large numbers | Dark text | `#1E2E1E` | Big stat numbers — NOT your secondary color. Dark text for big numbers. |

**Hard rules (apply regardless of your brand colors):**
- Accent color is used ONCE per image maximum. It is a signal, not a theme.
- The background should be a light, neutral tone. Never white. Never dark.
- Maximum 3-4 colors visible in any single image: background + primary + secondary + one accent.
- **Large numbers and key stats use dark text, NOT your secondary color.** Light colors on light backgrounds lack contrast at large sizes.

## Typography Style

The look: vintage-modern editorial. Bold italic serif for headings. Clean lighter weight for body.

| Role | Style | Minimum size (for mobile readability) |
|---|---|---|
| Hero headline | Bold italic serif (default: Playfair Display) | 50px minimum, prefer 60-76px |
| Section labels | Bold serif or bold sans-serif, medium weight | 26px minimum |
| Captions / step labels | ALL CAPS, spaced letter-tracking | 14px minimum |
| Data numbers (stats) | Extra-large bold serif — the biggest element | 80px minimum |
| Body / descriptions | Clean regular weight (default: DM Sans), readable | 18px minimum |
| Detail text (context, subtitles) | Regular weight | 28px minimum for important context |

**Phone readability is critical.** LinkedIn is predominantly viewed on mobile. If text can't be read on a phone screen without zooming, it's too small.

## Headline Rules — Scroll-Stopping Required

The image headline is the single most important line. It must stop a fast scroll.

**The test:** Would someone who sees only this image — no post text — feel like it might be about them or their situation? If yes, it stops the scroll.

**Rules for headlines:**
- Address the reader directly when possible — "your", "you", "most people like you"
- Lead with the failure/risk/cost, not the solution — people stop for pain more than gain
- Use a specific number if the post has one — numbers in headlines outperform vague claims
- The headline must work as a standalone statement — no post text needed to understand it
- **The Social Media Designer Agent must rewrite the post's opening line into a scroll-stopping headline.** Do not just pull the first sentence of the draft.
- **NEVER target specific roles or people in headlines.** See `rules/content-boundaries.md`.

---

## Verdict Labels — Required on All Comparison Images

Any image with two sides, two options, or a contrast structure MUST have a verdict label on each side.

| Situation | Left (bad) label | Right (good) label |
|---|---|---|
| Failure vs success | "THE WRONG WAY" | "THE RIGHT WAY" |
| Most vs few | "WHY MOST FAIL" | "WHY 10% WIN" |
| Old vs new | "THE OLD WAY" | "THE BETTER WAY" |
| Problem vs solution | "THE MISTAKE" | "THE FIX" |

---

## Context Anchor — Required on All Stat Images

Every stat or number in an image must have a one-line context anchor below it in ALL CAPS text at 15px+ minimum. It answers: this number is about *what*?

The bold_text_metaphor template has a `metaphor_html` field. Use concrete visual metaphors:

| Concept | Visual Metaphor Options |

## DO NOT Patterns

1. **Secondary color as large text color.** Light colors on light backgrounds lack contrast at large sizes. Use dark text for big numbers and headlines.
2. **Author name on every image.** Keep it clean. Author info goes in the LinkedIn post itself.
3. **Text too small for mobile.** If it can't be read on a phone without zooming, it's too small.
4. **Headlines that target specific roles.** Never "Middle managers who..." or "Team leads who...". Blame systems and processes, not people. See `rules/content-boundaries.md`.
5. **Passive headlines.** "The difference isn't technology" doesn't stop a scroll. "They compressed 9 months into 30 minutes" does.
6. **Oversized file output.** Template JPGs should be 150-300KB. If larger, check for unnecessary complexity.
