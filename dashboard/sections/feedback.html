System / Feedback Loops

Self-Improving System

The system doesn't just execute — it learns. Every human interaction feeds back into the pipeline through the Marketing Director, gradually improving both content quality and design output over time. Three Telegram commands are the entry points.

LOOPS: 2
ENTRY POINTS: 3 COMMANDS
CYCLE: WEEKLY
PROCESSOR: FT.02 MARKETING DIRECTOR
LOOP_01 — CONTENT QUALITY
ACTIVE
01
Human Reviews Draft
Operator reads post text on Telegram. Sends edit notes, rejects, or provides strategic direction about content quality and approach.
/marketing_director
→
02
Feedback Logged
Edit notes, idea rejections, and /marketing_director messages are all saved to agent_logs with distinct action tags so they can be filtered and analyzed.
agent_logs → action: edit_requested
agent_logs → action: idea_adjustment
agent_logs → action: operator_feedback
04
Downstream Agents Read
Content Strategist generates ideas aligned with updated direction. Content Creator writes posts following new patterns. Quality improves incrementally each week.
FT.03 reads performance-patterns.md
FT.04 reads performance-patterns.md
←
03
Marketing Director Synthesizes
Weekly, FT.02 reads all feedback from the week. Identifies patterns — repeated rejections, edit themes, strategic preferences — and updates the direction file.
→ performance-patterns.md
→ content-gaps.md
↻ CYCLE REPEATS WEEKLY
LOOP_02 — DESIGN QUALITY
ACTIVE
01
Human Reviews Design
Operator sees the generated design on Telegram. Logs quality observations — layout clarity, text readability, visual balance, template fit — without triggering regeneration.
/design_feedback [notes]
→
02
Feedback Logged
Design quality notes saved to agent_logs with a distinct tag. Regeneration feedback also saved separately. Both are readable by the Marketing Director.
agent_logs → action: design_quality_feedback
agent_logs → action: design_feedback
04
Designer Reads Updated Rules
Next time FT.06 generates a design, it loads the updated brand-guidelines-skill.md. The operator's feedback is now baked into the visual direction. Designs improve over time.
FT.06 reads brand-guidelines-skill.md
←
03
Marketing Director Analyzes Patterns
Weekly synthesis. If 3+ design feedback notes point the same direction (e.g. "text too small"), FT.02 adds an Operator Feedback Patterns section to the brand guidelines file.
→ brand-guidelines-skill.md
→ performance-patterns.md (design section)
↻ CYCLE REPEATS WEEKLY
IMMEDIATE ACTION — DESIGN REGENERATION
INSTANT
01
Operator Sends Feedback
Not happy with the current design. Sends specific visual feedback directly on Telegram.
/regenerate_design [feedback]
→
02
Designer Regenerates
FT.06 fires immediately with the feedback injected. Adjusts template parameters incorporating the notes, regenerates the design, delivers the new image to Telegram. Feedback also logged for the weekly loop.
FT.06 → Template Engine → Telegram
