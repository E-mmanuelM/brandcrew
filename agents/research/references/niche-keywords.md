# Research Agent — Niche Keywords Reference

> Loaded by Research Agent when building search queries.
> Full niche context in `config/niche.md` — always load that file first.

This file provides search guidance for the Research Agent. The actual keywords
come from `config/niche.md` — the user defines their niche there.

---

## How to Build Search Queries

1. Read `config/niche.md` for the primary keywords, tools, and topic areas
2. Combine keywords with the search patterns below
3. Prioritize practitioner content over vendor/consultant content

## Primary Search Patterns
Use these patterns with the keywords from config/niche.md:

- `[keyword] + tutorial`
- `[keyword] + step by step`
- `[keyword] + how to`
- `[keyword] + automation`
- `[keyword] + implementation guide`
- `[keyword] + 2026`
- `[tool from niche.md] + for beginners`
- `[tool from niche.md] + real experience`
- `[industry from niche.md] + AI tools`
- `[industry from niche.md] + process improvement`

## High-Value Content Signals
- Case studies with real numbers
- Step-by-step tutorials and implementation walkthroughs
- Tool launches or major updates in the niche's technology
- LinkedIn posts from practitioners with 500+ reactions
- Industry reports from major consulting firms on the niche
- "I tested this" or "here's what happened" practitioner posts
- Build-in-public implementation logs
- Beginner-friendly explainers for non-technical professionals

## Avoid
- Generic "AI is the future" articles with no practical angle
- Consumer-facing content (B2C, retail, lifestyle) unless niche is consumer
- Topics already covered in last 30 days (check Supabase)
- Vendor press releases without practitioner perspective
- Mega-influencer content (500k+ followers) — not comparable to practitioner scale
